Quickstart

Minimal app flow

`bruvtext` is meant to feel like a normal text library from app code. Create one context, register fonts, begin a frame, queue text, end the frame, then let the renderer consume atlas pages and draw batches.

CMake

Add the library target

add_subdirectory(third_party/bruvtext)
target_link_libraries(my_app PRIVATE bruvtext)

App Code

Queue text each frame

bruvtext::ContextDesc desc{};
desc.assetRoot = "third_party/bruvtext";

bruvtext::Context* text = bruvtext::CreateContext(desc);
bruvtext::FontId font = bruvtext::RegisterFont(
    text, "assets/fonts/YourFont.ttf", "UI");

bruvtext::BeginFrame(text);
bruvtext::DrawText(
    text,
    font,
    "Hello",
    32.0f,
    64.0f,
    20.0f,
    bruvtext::Color{1.0f, 1.0f, 1.0f, 1.0f});
bruvtext::EndFrame(text);

Measurements

Use the library for layout math

bruvtext::Vec2 size =
    bruvtext::MeasureText(text, font, "Hello", 20.0f);

float lineAdvance =
    bruvtext::MeasureLineAdvance(text, font, 20.0f);
`MeasureText` and `MeasureLineAdvance` are enough for basic centering, spacing, and line layout in caller code. The library does not try to own higher-level page layout.

Next

After `EndFrame`

  1. Inspect atlas pages from the context.
  2. Upload dirty pages to GPU textures.
  3. Read draw glyphs or draw batches.
  4. Submit the text draw pass in your renderer.

The exact renderer split is documented on the renderer contract page.
