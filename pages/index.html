FreeType + HarfBuzz + Vulkan

bruvtext

Boring Rudimentary Vulkan Text. A vendorable text library for Vulkan projects with a normal draw/measure API, cached glyph atlases, and a reference GPU renderer.

GitHub README Quickstart

What It Is

  • Vendored library, not a UI framework.
  • Shape with HarfBuzz, rasterize with FreeType.
  • Render from cached glyph atlases on the GPU.
  • Ship your own fonts; demo fonts stay demo-only.

API

DrawText

Simple draw, explicit draw-ex, and measurement without public cache knobs.

Coverage

19 languages

Latin, CJK, Arabic, and Devanagari demo coverage in the bundled showcase.

Quick Start

CMake

add_subdirectory(third_party/bruvtext)
target_link_libraries(my_app PRIVATE bruvtext)

App code

bruvtext::BeginFrame(text);
bruvtext::DrawText(text, font, "hello", 32.0f, 64.0f, 20.0f);
bruvtext::EndFrame(text);

Ownership Split

bruvtext owns

  • font shaping
  • glyph caches
  • atlas page contents
  • draw glyph / draw batch data

Host renderer owns

  • Vulkan images and buffers
  • atlas uploads
  • descriptor sets
  • draw submission

Current Shape

  • Raylib-style public API with draw and measure.
  • Exact-size glyph buckets with lazy cache growth.
  • Internal shaped-run cache for repeated strings.
  • Reference benchmark and showcase inside the bundled demo.

What It Is Not

  • Not a layout engine.
  • Not a rich text editor.
  • Not a mandatory font pack.
  • Not a framework that takes over your renderer.
