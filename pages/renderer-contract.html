Renderer Contract

Who owns what

The library does not take over Vulkan. It generates text data for a host renderer. The demo renderer is a reference implementation of that contract.

Ownership Split

Explicit responsibilities

bruvtext owns

  • font loading
  • HarfBuzz shaping
  • FreeType rasterization
  • glyph atlas contents
  • draw glyph and draw batch generation

Host renderer owns

  • Vulkan images and samplers
  • buffer allocation
  • descriptor sets
  • pipeline state
  • draw submission

Per Frame

Typical integration flow

  1. Call BeginFrame.
  2. Queue text with DrawText or DrawTextEx.
  3. Call EndFrame.
  4. Upload dirty atlas pages.
  5. Build or upload renderer vertex data.
  6. Submit the text pass.

Flexibility

What this preserves

  • screen-space HUD and menu text
  • world-space labels on 3D quads
  • document rendering
  • render-to-texture passes

The library gives you shaped 2D glyph layout and atlas-backed draw data. The host can reinterpret that into whatever final transform or rendering pass it needs.
